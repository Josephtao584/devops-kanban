## 5. 测试策略、关键测试用例与完成标准

### 测试策略
| 测试层级 | 工具 | 覆盖范围 |
|---------|------|---------|
| 单元测试 | Vitest (frontend) / Node test runner (backend) | Service 层业务逻辑、工具函数 |
| 集成测试 | Node test runner + tsx | API 端点、Repository 数据操作 |
| 端到端测试 | 手动验证 / 自动化脚本 | 完整用户流程 |

### 关键测试用例（模板）
| 用例编号 | 测试场景 | 输入 | 预期输出 |
|---------|---------|------|---------|
| TC-1 | 正常流程 | 有效请求参数 | 200 OK，返回正确数据 |
| TC-2 | 参数校验失败 | 缺失必填字段 | 400 Bad Request，错误信息明确 |
| TC-3 | 资源不存在 | 无效 ID | 404 Not Found |
| TC-4 | 并发写入 | 同时发起多个写请求 | 数据一致，无丢失 |

### 完成标准
1. 所有新增/修改的代码通过 lint 和 typecheck
2. 单元测试覆盖率不低于现有模块平均水平
3. 集成测试覆盖核心 API 端点
4. 手动验证完整用户流程无阻塞
