## 第1页：AI驱动的产业变革：与织女工的命运重演

**类比：工业革命 vs AI革命**

| | 工业革命 | AI革命 |
|---|---|---|
| **生产单元** | 织女工 → 动力织布机 | 程序员 → AI Agent |
| **组织形态** | 手工作坊 → 工厂 | 职能型开发团队 → 人机混合团队 |
| **管理对象** | 管人 → 管机器和流程 | 管工程师 → 编排Agent工作流 |
| **新岗位诞生** | 机器操作工、质检员、车间主任 | Agent监督者（Agent Supervisor）、AI编排者（AI Orchestrator）、AI运营（AI Operations） |

**组织变革的核心：**

- **个人层面**：程序员从"执行者"变为"指挥者"，核心能力从写代码转向意图描述、质量审查和流程编排
- **团队层面**：团队构成从"全员写代码"变为"少数人编排AI + 多数Agent并行执行"，5人团队+10个Agent的产出超过传统20人团队
- **管理层层面**：管理重心从"盯人"转向"盯流" — 关注任务流转效率、Agent执行质量、瓶颈识别，而非个人工时
- **风险**：历史上的织女工砸过机器（卢德运动），今天的组织变革同样会遇到阻力 — AI不是来抢饭碗的，但**拒绝适应的人会被适应的人取代**

**核心论点：** 工业革命没有消灭纺织业，而是催生了现代制造业体系。AI革命不会消灭软件行业，而是催生**AI-Native的组织形态**。

**配图描述：**
> 一幅充满冲击力的左右对比画面。左边是工业革命时期：一群织女工站在手工织布机前，脸上带着焦虑和不安，她们身后是巨大的蒸汽动力织布机正在轰鸣运转，几个织女工已经被挤到角落里，手里的梭子掉落在地。右边是AI时代的镜像：一群程序员坐在电脑前，同样神情焦虑，他们身后是成排的AI Agent终端在自动运行，光标飞速跳动、代码自动生成，几个程序员已经推开椅子站了起来，茫然地看着屏幕。画面中央是一道闪电般的分割线，两侧形成对称构图 — 两个时代、两种工人、同一种命运：你的手艺不够用了。整体色调压抑但有张力，左边暖黄渐暗，右边冷蓝渐亮，暗示旧时代的终结和新世界的压迫感。

## 第2页：AI-First 的本质：全流程革命，重新定义人机分工

**开场衔接：**
> 第1页结论：AI革命催生AI-Native的组织形态。但现实是，大多数组织的"AI变革"只是给旧流程贴了一层AI的皮。

**AI-assisted 的现状：**
- 工程师开 Claude Code、产品用 ChatGPT 写 PRD、QA 试试 AI 生成用例
- 流程没变、Sprint 没变、评审没变 — 老流程里塞进新工具
- 效率提升 10-20%，但结构本身没有变化
- 就像织女工换了一把更快的梭子 — 速度变了，生产方式没变

**AI-First：重新定义人机分工：**
- 不是问"AI怎么帮助工程师"，而是 — **如果默认 AI 是主要建设者，流程、架构、组织该怎么重构？**
- 执行者变了：人不再是每个环节的执行者，而是设计者
- 交接方式变了：人传人的信息损耗，变成 Workflow 自动流转
- 对应织女工的类比：不是换梭子，而是从手工织布变成管动力织布机

**瓶颈已经转移：**
- AI 两小时实现一个功能，拖慢团队的不再是开发
- PM 瓶颈：Agent 两小时写完功能，产品规划要几周
- QA 瓶颈：AI 实现完，验证速度跟不上实现速度
- 人头瓶颈：团队规模不可能靠招人追平对手
- 只要有一条链路还是手工的，就会卡住整个系统

**引出第3页：**
> 那全流程革命具体怎么落地？我们需要一套工程实践来回答。

**配图描述：**
> 一幅对比示意图。左半边是一个传统软件开发流水线，人（小人图标）在每个环节手动传递文档，流程缓慢、中间有瓶颈（用沙漏图标表示卡顿点）。右半边是AI驱动的全新流水线，AI Agent（发光的机器人图标）在各环节自动执行，人只站在几个关键决策点（用放大镜图标表示），流程流畅无阻。左右之间用一道裂缝隔开，象征范式转变。科技感扁平插画风格。

## 第3页：下一代工程师桌面：一个人就是一支团队

**类比：**
- 从前：织女工坐在一台织布机前，一针一线亲手织
- 现在：工厂里十台动力织布机同时运转，她站在车间里巡视、调参数、验质量

工程师的转变一样：
- 从前：一个人坐在编辑器前，一行行亲手写代码
- 现在：你身后站着一群 Agent，编码的、测试的、审查的 — 一个人就是一支全功能团队

**一天的工作变了：**
- 早上到岗，桌面告诉你：昨夜 6 个任务完成、2 个等你确认、1 个需要你的判断
- 你的动作：审查产出 → 给反馈或批准 → 描述新任务 → 回去喝咖啡
- 不再是你追着工作跑，而是**工作做完来找你**

**核心体验：**
- **可见** — 所有进展实时呈现，不需要开会问进度
- **可控** — 关键节点等你拍板，Agent 不会擅自行动
- **可介入** — 方向不对随时喊停、纠正、重来

**一句话：** 以前一个人只能干一个人的活，现在一个人指挥一群 Agent — **你的上限不再是你个人的产能，而是你编排和管理的能力**。

**配图描述：**
> 一个工程师坐在宽大的办公桌前，神态从容放松，手里端着咖啡。他面前的屏幕上是一个简洁的仪表盘界面，显示多个任务卡片在不同状态（进行中、等待确认、已完成）。他身后整齐排列着6-8个半透明的发光AI助手人形，各自在不同工位上忙碌工作（有的在写代码、有的在测试、有的在审查），但都和工程师通过光带连接。整体传达"一个人指挥一群AI"的感觉。未来科技办公风格，明亮干净。

## 第4页：AI-Native桌面的能力底座：感知、编排、观测、控制

**核心内容：**

> 一个人指挥一群Agent干活，桌面必须回答三个问题：Agent怎么知道该干什么、人怎么知道Agent干得怎样、出了问题人怎么管。

**需求感知能力**
- 外部需求自动同步，不用人工搬运
- 需求自动拆解为可执行任务，每个任务携带完整上下文
- Agent 拿到任务就知道做什么、做成什么样，不需要人反复解释

**编排调度能力**
- 任务自动流转 — 编码完进入测试，测试完进入审查，不需要人催进度
- 不同环节交给最合适的 Agent，各展所长
- 多个 Agent 并行执行不同任务，互不干扰

**透明可观测能力**
- Agent 在想什么、调了什么、产出了什么，全程实时可见
- 不是事后看报告，而是像看直播一样看到执行过程
- 卡住了、报错了，第一时间感知

**人在回路能力**
- 关键节点自动暂停等人确认，不擅自行动
- 方向不对随时叫停、纠正、重来
- 审查通过才放行，不通过打回去重做

**一句话：** 感知、编排、观测、控制 — 四种能力让一个人真正管得住一群Agent，**不是放出去不管，而是全程在掌控之中**。

**配图描述：**
> 一个四象限的概念图，中心是一个发光的桌面/屏幕图标。四个象限分别用图标+简短文字表示：左上"感知"（雷达/天线图标，数据流入），右上"编排"（齿轮/管道图标，任务自动流转），左下"观测"（眼睛/监控屏图标，实时透明），右下"控制"（手/开关图标，人做决策）。每个象限之间有箭头连接形成闭环。四象限外围是一圈Agent图标，表示它们被这四种能力统一管理。科技感信息图风格，深色背景配蓝色/青色发光线条。

## 第5页：DevOps Kanban：从能力底座到可靠交付的 Harness 实践

**核心内容：**

> Agent 跑一次 demo 很惊艳，但放到生产环境，让 AI 的产出可靠可用，比让它干活难十倍。DevOps Kanban 的核心是用 Harness 工程纪律让 Agent 从"能用"变成"可控"。

**四个工程实践：**

**① 上下文架构 — 让 Agent 知道该做什么**
- 不是丢一句"实现这个功能"，而是给 Agent 完整上下文 — 需求描述、验收标准、依赖关系
- 任务源自动同步，任务卡片承载完整信息，Workflow 模板预置上下文传递规则
- 上游步骤的摘要自动传递给下游，Agent 之间不需要人反复解释

**② Agent 专业化 — 让最合适的 Agent 做最合适的事**
- 编码阶段用 Claude Code，验证阶段用 OpenCode，审查阶段用 AI 并行三轮审查
- 不是一个大模型干所有事，而是按环节分配最擅长的执行者

**③ 持久化记忆 — 让每一步都有交付件，每次执行都有积累**
- 每个步骤完成后，执行摘要、产出物、代码变更自动归档
- 执行断了不用重头来 — 从失败步骤重试，上下文链完整保留
- 完整执行过程可回放

**④ 结构化执行 — 给 Agent 划定边界**
- 每个任务在独立 Git Worktree 中执行，多 Agent 并行互不干扰，失败不影响主分支
- 关键节点自动暂停等人确认，确认后跳过已完成步骤继续
- 审查不通过打回重做，通过才放行

**一句话：** Harness 的本质是**给 AI 套上缰绳** — 不是限制能力，而是让能力在可控的轨道上释放。

**配图描述：**
> 一个"缰绳"的隐喻视觉：画面中心是一匹充满力量的骏马（代表AI Agent），身上套着精致的蓝色发光缰绳和马鞍（代表Harness工程纪律），缰绳握在一个工程师手中。工程师不是在拉住马，而是轻轻引导方向。马在一条明亮的轨道上奔跑，轨道两侧有护栏和检查站（代表上下文架构、持久化记忆、结构化执行等工程实践）。远处是生产环境的目标。整体传达"不是限制，而是可控地释放力量"。科技感数字插画风格。
