## 第1页：从 Harness 到落地：AI-Native 的下一个挑战

**布局：左图右文**

**正文：**

Coplat 已经实现了四个 Harness 实践——上下文架构、Agent 专业化、持久化记忆、结构化执行。这是 AI-Native 的执行底座。

但底座可用和团队落地之间，还有四个核心挑战：

- **安全**：Agent 可能做不该做的事——推远端、触发 CI、操作生产环境
- **监控**：Agent 执行过程不够透明——卡在哪、为什么卡、要不要介入
- **质量**：Agent 产出可能不对——"看起来对但实际错"，审查跟不上产出速度
- **效率**：Agent 执行有成本——token 消耗、重试成本、等待时间

四个挑战的根因是同一个：**Agent 执行的不确定性**。固定脚本跑错了会报红，Agent 跑错了可能不报错。

**一句话：** Harness 实践保证了 Agent 产出质量，安全、监控、质量、效率保证 Agent 行为可控、过程透明、产出正确、成本可接受。

## 第2页：挑战一·安全 — Agent 行为边界

**布局：上文下图**

**正文：**

Agent 在 Worktree 内写代码是安全的。Worktree 隔离保护了主分支。但 Agent 的能力可以延伸到 Worktree 之外：

| Agent 能做的 | 风险 |
|------------|------|
| `git push` 推送到远端 | 未经审查的代码进入远端 |
| `gh workflow run` 触发 CI | 绕过审查触发流水线 |
| `curl` 调用外部 webhook | 触发部署等外部变更 |
| 修改 CI 配置文件 | 改变团队交付流程 |
| `npm publish` / `docker push` | 直接发布到生产环境 |

**应对措施：**
- 工具级别执行限制——每个 Agent 配置工具权限，禁止高风险命令
- 凭证隔离——Agent 环境不注入生产 token 和密钥
- CI/CD 触发权归平台——合并、CI 触发、部署由平台在人确认后执行，不走 Agent
- 敏感文件保护——CI 配置文件对 Agent 默认只读

**一句话：** Agent 负责写代码，平台负责交付。触发权归平台不归 Agent。

**配图（1920×500px）：**

> 横向对比图。左半边"Agent 的安全边界"：一个 Worktree 目录的围栏（绿色发光），Agent（小人图标）在围栏内安全地写代码。右半边"边界之外的风险"：围栏外面是远端仓库（GitHub 图标）、CI/CD 流水线（齿轮图标）、生产环境（服务器图标），Agent 伸出红色虚线箭头试图触碰这些外部系统，箭头被一个红色的"X"挡住。中间是一道明显的边界线，标注"Agent 行为边界"。底部标注应对措施的关键词：工具限制 | 凭证隔离 | 平台接管 | 文件保护。扁平科技插画风格。

## 第3页：挑战二·监控 — Agent 执行过程可观测

**布局：左文右图**

**正文：**

固定脚本跑失败了会报错，Agent 跑失败了可能什么都没产出就结束了。你不知道它卡在哪、为什么卡、要不要介入。

当前 Coplat 有 Session Event 三层记录，但这是事后日志，不是实时监控。

**编排者需要知道的：**
- Agent 正在做什么、卡在哪里、已经花了多长时间
- 执行异常——长时间无输出、重复失败、产出不完整
- 每个步骤的执行耗时和产出物

**需要做的：**
- 执行过程实时可观测——Agent 的每步操作实时展示在 Workflow Timeline
- 异常状态主动告警——超时、重复失败、产出不完整，平台主动推送
- 步骤产出物自动校验——交付物不存在，不标记为成功

**一句话：** 不是等 Agent 跑完再看结果，而是全程知道它在干什么。

**配图（800×700px）：**

> 一个实时监控仪表盘的概念图。画面主体是一个大屏幕，屏幕上显示多个任务卡片在水平时间线上推进，每个卡片标注状态（运行中/已完成/异常），运行中的卡片有脉冲动画效果。屏幕右侧是告警面板，一个红色告警条目正在闪烁，标注"步骤3 超时无输出"。一个工程师坐在屏幕前，手里拿着咖啡，表情从容——因为他能看到一切。底部是时间轴，标注每步的耗时柱状图。整体传达"全程透明、异常即时感知"。科技感 UI 插画风格，深色背景配青色发光。

## 第4页：挑战三·质量 — Agent 产出正确性

**布局：左右分割**

**左半边 — 影响质量的因素：**

- **上下文质量**：任务描述模糊 → Agent 拿到模糊需求 → 产出大概率不对
- **任务粒度**：大任务拆成3个，每个成功率80%，整体只有51%
- **Agent 匹配度**：通用 Agent 做所有事 < 专业 Agent 各做各的

**右半边 — 审查瓶颈：**

Agent 两小时写完功能，人要花多长时间审查 500 行 diff？

审查速度跟不上产出速度，瓶颈从开发产能转移到审查产能。

**应对措施：**
- 自动化门禁前置：lint + 类型检查 + 单元测试在人工审查前自动跑
- Agent 审 Agent：编码产出先过审查 Agent，标注风险点，人只看标注部分
- 变更风险评估：根据变更范围自动评级，低风险简化审查，高风险增加环节
- 执行结果统计：追踪哪些任务成功率高、哪些低，数据驱动优化

**一句话：** 机器能查的不让人看，人只看机器标出来的疑问。

**配图（左右各 600×500px）：**

左图（质量影响因素）：
> 三个上下堆叠的漏斗，从上到下标注"上下文质量""任务粒度""Agent 匹配度"。每个漏斗上方是大量的任务输入（蓝色小球），通过漏斗后只有一部分变成绿色（成功），其余变成红色（失败）。三个漏斗的质量依次递增，传达"三个因素层层过滤成功率"的概念。扁平信息图风格。

右图（审查瓶颈）：
> 一个传送带对比图。上半部分：Agent 产出（代码块）高速从左侧涌出，排成长队等待审查。右侧一个工程师图标，头上冒汗，面前堆满了待审查的代码。下半部分（加了应对措施后）：同样的 Agent 产出，但先经过一个"自动门禁"过滤器（齿轮图标），过滤掉大部分问题；剩余的经过"审查 Agent"（小机器人图标）预审标注；最终只有少量标注项到达工程师面前，工程师表情从容。传达"机器过滤+人做判断"。扁平插画风格。

## 第5页：挑战四·效率 — Agent 执行成本

**布局：上文下图**

**正文：**

Agent 执行有直接成本和间接成本：

| 成本类型 | 具体表现 |
|---------|---------|
| **Token 消耗** | 每个步骤消耗 token，复杂任务可能数十万；多任务并行时线性增长 |
| **重试成本** | 失败重试消耗额外 token 和时间 |
| **等待成本** | 执行时间不可控，人在等待中空闲 |
| **审查成本** | 审查 500 行 diff 可能比 Agent 写代码的时间更长 |

**需要做的：**
- 上下文传递优化：上游摘要控制长度，不把原始输出塞进下一步 prompt
- 并行资源调度：同时运行的 Agent 有上限，平台做队列和调度
- 执行成本可视化：每个任务、每个步骤的 token 和时间消耗可追踪

**一句话：** 知道成本在哪，才能优化成本。

**配图（1920×500px）：**

> 一个横向的资源消耗分析面板。左侧是一个柱状图，横轴是任务步骤（编码/测试/审查），纵轴是 token 消耗量，柱子高度不一，编码步骤最高。右侧是并行任务的时间线图，5个任务水平排列，有的并行运行，有的排队等待（灰色），标注"并发上限=3"。底部是一个总成本汇总条，显示"Token 总消耗 / 执行总时长 / 重试次数"。一个工程师站在面板前分析数据，手指向最高的柱子。传达"成本可见、可追踪、可优化"。科技感数据可视化插画风格。

## 第6页：落地路径 — 从 demo 到团队长期运行

**布局：三列并排**

**正文：**

**阶段一（近期）— 利用现有机制，不改代码**

- 为所有 Agent 配置 `disallowedTools` settings 文件
- Workflow 模板规范：CI/CD 触发不写进 instructionPrompt
- 步骤超时和重试策略
- 执行过程实时可观测基础建设
- 自动化门禁前置：lint + 类型检查 + 单元测试

**阶段二（中期）— 平台能力建设**

- Agent 配置页增加工具权限 UI
- CI/CD 触发原生步骤类型，对接 GitHub Actions API
- 产出完整性校验 + 异常状态主动告警
- Agent 审 Agent（编码 → 审查 Agent 预审 → 人工确认）
- 执行结果统计和成本可视化

**阶段三（长期）— 完整治理闭环**

- CI 状态回流 Workflow Timeline，完整链路可追溯
- 凭证隔离 + 敏感文件保护
- 变更风险评估和分级审查
- 并行执行的资源调度和队列管理

**底部一句话：** Harness 实践是基础，安全、监控、质量、效率的治理是进阶。两者加在一起，才是从"能跑 demo"到"能在团队里长期运行"的完整路径。

**配图：**

> 不用 AI 生图。三列并排的路线图，每列是一个阶段，用不同颜色区分（阶段一蓝色、阶段二绿色、阶段三金色）。每列内部是垂直排列的任务卡片，卡片上有关键词和对应挑战维度的彩色标签（安全=红色、监控=青色、质量=黄色、效率=蓝色）。三列之间用向右的箭头连接，箭头上标注关键门槛（"不改代码"→"平台能力"→"治理闭环"）。底部是总结横条。扁平信息图风格，适合直接在 PPT 中排版。
