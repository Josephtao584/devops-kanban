设计要点

核心交互

  • 混合信息流 — 用户消息、AI 规划、Agent 执行动态、任务完成通知按时间线交织
  • 底部对话框 — 用户输入指令，AI 的回复和操作都体现在流中
  • 嵌入式任务卡片 — 任务不再是独立的面板，而是信息流中的卡片元素
  • 实时 Agent 状态 — 看到 Agent 正在做什么，带打字动画指示

与现有视图的关系

  • 看板视图 — 保留，适合宏观总览和手动拖拽管理
  • 列表视图 — 保留，适合批量编辑和筛选
  • AI 视图（新） — 适合 AI 驱动的工作模式，用户通过对话推进项目
  • 三种视图共享同一套任务数据，切换视图不丢失状态
