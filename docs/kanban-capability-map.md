```
╔══════════════════════════════════════════════════════════════════════════════════════════════════════
║  DevOps Kanban 能力全景图
║
║  核心理念
║  ────────
║  让 AI Agent 成为可靠的生产力
║  Harness Engineering —— 为 AI Agent 构建工程纪律，让 Agent 在任何场景下稳定交付结果
║
║  AI 辅助 vs AI First
║  ─────────────────────
║  AI 辅助: 在旧有流程(编码/PRD/测试)中引入 AI → 效率提升 10-20%，瓶颈和结构未变
║  AI First: AI 是默认执行者，人类提供方向/判断/约束 → 全流程重构，瓶颈转移后持续突破
║
║  Kanban 的定位: 为 AI Agent 构建工程纪律，让 AI First 从口号变成可操作的 Harness 系统
║  ─────────────────────────────────────────────────────────────────────────────
║  AI First 带来的瓶颈转移与 Harness 应对（按软件研发周期顺序）:
║
║  需求进来了没人管         → Harness: 外部源同步 + Epic 拆解 + 需求澄清
║  需求拆不成可执行任务     → Harness: 任务拆解规则 + AI 辅助分解
║  任务来了等人工分配启动   → Harness: 自动调度引擎（定时同步/自动匹配/自动启动/自动通知）
║  多任务同时跑互相冲突     → Harness: Worktree 隔离 + 冲突检测 + 写队列串行化
║  做完没人验证质量         → Harness: 自动测试 + 质量扫描 + 用例生成
║  产出太多审查看不过来     → Harness: AI 审查 + 安全扫描 + 规范检查
║  上线后问题发现到修复太慢 → Harness: 自愈闭环(监控→分诊→工单→修复→验证)
║
╠══════════════════════════════════════════════════════════════════════════════════════════════════════
║
║  Harness 四大支柱 —— AI First 不是放任 AI，而是让 AI 在工程框架内释放最大生产力
║
║  ① 上下文架构 Context Architecture
║  ──────────────────────────────────
║  为什么: AI 的输出质量取决于输入的上下文质量。零散的提示词产出不可预测，
║         结构化的上下文才能让 AI 稳定地产出工程级结果。
║  怎么做: Workflow 模板定义每一步的指令 + MCP/Skill 注入专业能力 + 任务卡片承载需求上下文
║  落地:   [✓] Workflow 模板  [✓] Skill/MCP 系统  [✓] 任务结构化卡片  [ ] 知识库/RAG
║
║  ② Agent 专业化 Agent Specialization
║  ──────────────────────────────────
║  为什么: 没有 Agent 能做好所有事。不同任务需要不同能力——设计、编码、测试、审查各有最佳 Agent。
║         专业化才能让每个环节都达到最高质量。
║  怎么做: 每个步骤独立分配 Agent，模板预置角色配置，开箱即用。
║  落地:   [✓] 多执行器(Claude Code/Codex/OpenCode)  [✓] 按步骤分配 Agent  [ ] Agent 角色推荐
║
║  ③ 持久化记忆 Persistent Memory
║  ──────────────────────────────────
║  为什么: AI 没有记忆就没有积累——每次从零开始是最昂贵的浪费。
║         记忆让 AI 能从失败中学习，让历史经验成为下一次的起点。
║  怎么做: Session/Segment 三层模型保留完整执行链，Event 按序持久化，Template Snapshot 冻结运行时状态。
║  落地:   [✓] Session/Segment/Event 三层  [✓] 断点续接  [✓] 模板快照  [ ] Agent 记忆系统  [ ] 知识库/RAG
║
║  ④ 结构化执行 Structured Execution
║  ──────────────────────────────────
║  为什么: AI 在野环境里横冲直撞是灾难。结构化执行给 AI 划定边界——
║         隔离的执行环境、确定性的流程、人工介入的安全阀。
║  怎么做: Git Worktree 隔离 + Workflow 步骤编排 + Suspend/Resume 人工确认 + 写队列防竞态。
║  落地:   [✓] Worktree 隔离  [✓] Workflow 编排  [✓] Suspend/Resume  [✓] 写队列串行化  [ ] CI/CD 触发
```
