# 需求分析 Prompt

将 `service-analysis/architecture.yaml` 的内容粘贴到下方 `{架构全景}` 处，然后把整段发给 LLM。

---

```
以下是我们的微服务架构全景（YAML 格式）：

{架构全景}

---

新需求：{在这里描述你的需求}

请分析：

1. **涉及的服务**：列出所有需要改动的服务，每个服务说明需要做什么
2. **调用链路**：请求从哪个服务进来，经过哪些服务，数据怎么流转
3. **接口变更**：是否需要新增或修改服务间的接口
4. **数据模型变更**：是否涉及数据库表或前端状态的变更
5. **关联影响**：有没有可能遗漏的关联服务

输出格式：按服务分组，每个服务列出具体改动点和工作量评估（小/中/大）。
```
